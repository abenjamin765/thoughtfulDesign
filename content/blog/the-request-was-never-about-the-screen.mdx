A team asks you to redesign a dashboard. The brief is one sentence: make it easier
to use. It sounds like a layout problem — tighten the hierarchy, cut a few charts,
ship a cleaner screen.

So you do. And three weeks later, the same complaints come back in slightly
different words.

That's the tell. When a clearer screen doesn't fix the confusion, the confusion was
never on the screen.

## What the request is hiding

"Easier to use" is a symptom. Underneath it, there are usually four quieter
questions no one has answered out loud:

- **What is this for?** Different stakeholders are quietly optimizing for different
  outcomes.
- **Which things matter?** The objects the product is really about — the student, the
  order, the case — haven't been named, so every screen treats them inconsistently.
- **Who is this for?** The people using it have different goals, and the design serves
  an average of all of them.
- **Do we trust the data?** No interface can make untrusted numbers feel
  decision-ready.

None of those are solved by moving a chart.

## The shift

> The team asked for a better screen.
> The user needed a clearer system.

This is the move at the center of thoughtful design. Before you decide what should
appear, you understand what already exists: the system the screen belongs to, the
objects it's built from, and the decisions it's supposed to support.

It's slower at the start. It's much faster than redesigning the same dashboard three
times.

## A better first response

Next time a request arrives pre-solved as a screen, try trading the layout question
for a system question:

- Instead of *"What should this dashboard show?"* ask *"What decision is this
  dashboard supposed to support, and for whom?"*
- Instead of *"Which charts do we keep?"* ask *"Which objects does this product
  actually revolve around?"*

You're not refusing the work. You're making sure the work solves the real problem —
which is the most useful thing a designer can do.

That instinct — to look past the surface request to the system underneath — is where
the course begins.
