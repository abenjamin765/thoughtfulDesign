## The situation

The marketplace listing flow is reordered and its states are designed. But sellers still hesitate on the category step, where the interface presents a wall of forty subcategories and asks them to type a code from a help article. Every choice feels like a small test.

## Cognitive load is the hidden tax

Cognitive load is the mental effort an interface demands. Users have a limited budget of attention, and every unnecessary decision, memory task, or ambiguous label spends it. Reduce the load and the same task suddenly feels easy.

A few reliable levers:

**Recognition over recall.** Let people choose from what is shown rather than remember it. A labeled category list beats a memorized code every time.

**Fewer, clearer choices (Hick's Law).** The more options you present at once, the longer decisions take. Group, prioritize, or stage choices instead of dumping them all on one screen.

**Easy targets (Fitts's Law).** Make important controls large enough and close enough to where the user is looking. A tiny "list item" button in a far corner adds friction to your most important action.

**Progressive disclosure.** Show the simple path by default; reveal advanced options only when they are relevant. Most sellers never need bulk-listing settings on their first item.

## Clarity is craft, including the words

UX writing is part of interaction craft. A label like "Payout cadence" taxes a first-time seller; "How often you get paid" does not. Plain, specific words reduce load as much as layout does.

> Every unnecessary decision you remove is attention the user gets to spend on their actual goal.

<Callout type="tip">

Audit a screen by counting the decisions it forces. Then ask of each: can this be defaulted, deferred, grouped, or removed? Most overloaded screens shrink dramatically under that question.

</Callout>

<Callout type="warning">

A common trap is "simplifying" by hiding essential actions. Reducing load means removing *unnecessary* effort, not burying the things users came to do. Progressive disclosure hides the rare, not the core.

</Callout>

<Reveal label="What good looks like">

The marketplace team replaces the code field with a searchable, labeled category picker (recognition), shows the eight most common categories first with "more" expanding the rest (Hick's Law + progressive disclosure), enlarges the primary "Continue" button (Fitts's Law), and rewrites jargon labels in plain language. Sellers stop hesitating because each step now asks for one easy recognition, not a memory test.

</Reveal>
