## The bedrock of OOUX

OOUX is grounded in how people perceive the world. Four truths capture the most common ways digital systems break mental models.

<Callout type="truth" title="Ancient Truth #1 — Objects">

Things that are different should look different.

</Callout>

When objects are visually distinct, users build accurate mental models quickly. When different things look the same — or the same thing appears under different names — confusion follows.

<Callout type="truth" title="Ancient Truth #2 — Relationships">

Humans navigate most naturally through relationships between objects.

</Callout>

People do not think in nested menus. They think: "I'm looking at this BOOK, show me its LIBRARY" or "show me related EVENTs."

<Callout type="truth" title="Ancient Truth #3 — CTAs">

Humans act on objects through direct manipulation.

</Callout>

Actions should attach to the object they affect. If a user is viewing a TASK, they should be able to complete it right there — not hunt through settings.

<Callout type="truth" title="Ancient Truth #4 — Attributes">

Objects that are the same should behave the same.

</Callout>

Consistency does not mean identical layout everywhere. It means the same object type carries the same core attributes and behaviors across contexts.

## Common anti-patterns

| Truth | Anti-pattern |
| --- | --- |
| Objects | Masked objects — different things disguised as one |
| Relationships | Isolated objects — no visible connections |
| CTAs | Broken objects — visible but not actionable |
| Attributes | Shapeshifters — same object, inconsistent behavior |

<Reveal label="Quick memory hook">

Objects = recognition. Relationships = navigation. CTAs = action. Attributes = consistency.

</Reveal>
