# Ethical Risk Review

Run this on any feature before it ships, especially anything that touches money, defaults, consent, data, or behavioral nudges. The review only matters if it can change the design.

## The six questions

| Question | Your notes |
| --- | --- |
| Intended outcome — what is this for? | |
| Affected users — who uses it, especially the vulnerable? | |
| Possible harms — what could go wrong, and for whom? | |
| Power imbalance — what does the product control that the user doesn't? | |
| Missing transparency — what is hidden, defaulted, or hard to notice? | |
| Safer alternative — how do we keep the value and reduce the harm? | |

## The "say it out loud" test

Would the design survive being explained honestly to the user?
- Fails: "We pre-checked this so you'd be more likely to miss it."
- Passes: "We recommend this and here's why; it's easy to skip."

## Quality check

- [ ] Names a specific, plausible harm and a person it lands on.
- [ ] Acknowledges the power asymmetry honestly.
- [ ] Distinguishes persuasion (respects choice) from manipulation (subverts it).
- [ ] Ends with a concrete redesign, not a justification.
