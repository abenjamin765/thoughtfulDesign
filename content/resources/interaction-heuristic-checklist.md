# Interaction Design Heuristic Checklist

Use this to run a fast, structured critique of any screen or flow. For every issue you find, write the **principle**, the **specific instance**, and a **concrete fix** — then rank your top three.

## Review lenses

**Hierarchy**
- [ ] The most important element is the most prominent.
- [ ] Related items are grouped; unrelated items are separated.

**Flow**
- [ ] Steps follow the user's mindset, not the database.
- [ ] Motivation comes before friction (value before heavy asks).
- [ ] Each step asks for one clear thing.

**Copy**
- [ ] Labels use plain language, not jargon.
- [ ] Buttons name the action ("Publish listing," not "Submit").

**States**
- [ ] Empty state orients and offers the next action.
- [ ] Loading state shows progress; never freezes silently.
- [ ] Error state says what, where, and how to fix.
- [ ] Success state confirms and points to what's next.

**Feedback**
- [ ] Every action gets a timely, specific response.

**Effort (cognitive load)**
- [ ] Recognition over recall (choose, don't memorize).
- [ ] Choices are limited or staged (Hick's Law).
- [ ] Key targets are large and reachable (Fitts's Law).
- [ ] Advanced options use progressive disclosure.

## Quality check

- [ ] Every finding names a principle, not just a preference.
- [ ] Each has a concrete, buildable fix.
- [ ] The top three are ranked by user impact.
