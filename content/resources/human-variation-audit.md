# Human Variation Audit

Use this to turn "make it accessible" into a prioritized list of concrete changes. Walk the flow through each lens and ask: who could be excluded here, and how?

## Audit table

| Lens | Barrier in this flow | Who it affects | Concrete fix | Reach (people affected) |
| --- | --- | --- | --- | --- |
| Visual (contrast, color-only meaning) | | | | |
| Motor (target size, keyboard, hover) | | | | |
| Cognitive (jargon, memory, progress) | | | | |
| Emotional (stress, forgiveness of errors) | | | | |
| Linguistic (idioms, acronyms, translation) | | | | |
| Situational (glare, one hand, slow network) | | | | |
| Assistive tech (labels, structure, order) | | | | |

## Three fast checks

- [ ] Navigate the entire flow using only the keyboard.
- [ ] View it in grayscale — does any meaning disappear?
- [ ] Read every label aloud as if stressed and new to the domain.

## Prioritize and commit

- **Single change that helps the most people:** ____________________
- **Most severe total blocker (locks anyone out):** ____________________
- **Doing first this release:** ____________________

## Quality check

- [ ] Each barrier names a real person it affects.
- [ ] Each has a specific fix, not a vague aspiration.
- [ ] Total exclusions are treated as high priority regardless of group size.
- [ ] The audit ends in committed changes, not reassurance.
