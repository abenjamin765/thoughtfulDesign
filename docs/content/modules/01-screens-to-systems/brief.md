# Module 1 Brief — From Screens to Systems

> Locked brief. Source: requirements §6, Module 1. Do not invent scope beyond this.

- **Part:** 1 — Seeing the System
- **Module ID:** `01-screens-to-systems`
- **Core claim:** A screen is only the visible surface of a deeper system.
- **Loop step(s):** Model ("What exists?")
- **Recurring scenario:** Internal enterprise dashboard
- **Resource:** None
- **Public preview lesson:** `the-dashboard-request`

## Purpose

Introduce the central shift of the course: modern UX is not only about designing screens. It is about shaping systems.

## Learning objectives

Learners should be able to:

- explain the difference between screen-level and system-level design,
- identify hidden systems behind a product interface,
- describe how UX decisions affect people, teams, data, workflows, and behavior,
- recognize why modern UX requires broader design judgment.

## Key topics

- The evolution of UX
- Interface design vs. product design vs. systems design
- Design as decision-making
- Design as system behavior
- Hidden dependencies behind simple interfaces
- The role of the thoughtful designer

## Opening story structure

Open with a simple redesign request — "make this dashboard easier to use" — then reveal that the real problem involves unclear objects, conflicting stakeholder goals, poor data trust, and mismatched user needs.

## Inline check seeds

1. Which of the following is a screen-level problem?
2. Which of the following is a system-level problem?
3. A team asks for a new filter. What should a thoughtful designer ask first?
4. What hidden systems might sit behind a search results page?

## Module assessment focus

Scenario-based quiz where learners distinguish surface requests from deeper system problems.

## Lesson plan (3 lessons)

1. **the-dashboard-request** (preview) — the redesign request; surface vs. system tension; the Shift.
2. **surface-vs-system** — interface/product/systems design; hidden dependencies; design as decision-making.
3. **the-thoughtful-designer-role** — the role; Try It exercise; connect to Model step; bridge to Module 2.
