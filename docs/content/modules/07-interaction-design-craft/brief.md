# Module 7 Brief — Interaction Design Craft

> Locked brief. Source: requirements §6, Module 7. Do not invent scope beyond this.

- **Part:** 3 — Shaping Responsible Experiences
- **Module ID:** `07-interaction-design-craft`
- **Core claim:** Craft is where thoughtful decisions become visible.
- **Loop step(s):** Shape
- **Recurring scenario:** Marketplace onboarding flow
- **Resource:** Interaction Design Heuristic Checklist
- **Public preview lesson:** `it-works-but-it-feels-wrong`

## Purpose

Ground the course in practical UX execution. Thoughtful design must become clear, usable product experience.

## Learning objectives

Learners should be able to:

- apply core interaction design principles,
- improve clarity in flows and interfaces,
- design useful states and feedback,
- reduce cognitive load,
- improve forms, navigation, and content hierarchy,
- critique interfaces using evidence-based heuristics.

## Key topics

- Information architecture
- Navigation
- User flows
- Task flows
- Forms
- Feedback
- Empty states
- Error states
- Loading states
- Progressive disclosure
- Recognition vs. recall
- Fitts's Law
- Hick's Law
- UX writing
- Design systems

## Opening story structure

Open with a product that technically works but feels confusing, brittle, or effortful. Show how interaction craft turns system understanding into product clarity.

## Inline check seeds

1. Which button placement is likely easier to use?
2. Which error message is most helpful?
3. Which flow creates unnecessary cognitive load?
4. Which empty state best supports the user's next action?

## Guided practice

Learners critique a flawed interface and recommend improvements across: hierarchy, flow, copy, states, feedback, effort.

## Module assessment focus

Learners complete a heuristic review of a product scenario and select the most effective improvements.

3. **reducing-cognitive-load** — recognition vs. recall; Hick's/Fitts's; progressive disclosure; UX writing.
4. **critique-with-heuristics** — Try It with Heuristic Checklist; bridge to Module 8.
