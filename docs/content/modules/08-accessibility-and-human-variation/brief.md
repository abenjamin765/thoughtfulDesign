# Module 8 Brief — Accessibility and Human Variation

> Locked brief. Source: requirements §6, Module 8. Do not invent scope beyond this.

- **Part:** 3 — Shaping Responsible Experiences
- **Module ID:** `08-accessibility-and-human-variation`
- **Core claim:** Good design respects human variation.
- **Loop step(s):** Shape, Account
- **Recurring scenario:** Healthcare appointment portal
- **Resource:** Human Variation Audit
- **Public preview lesson:** `the-user-who-isnt-disabled`

## Purpose

Frame accessibility as a core measure of design quality and a way to design for real human variation.

## Learning objectives

Learners should be able to:

- explain accessibility beyond compliance,
- identify common accessibility barriers,
- recognize cognitive, motor, visual, auditory, linguistic, emotional, and situational needs,
- apply basic WCAG-informed thinking,
- design for stress, fatigue, distraction, and neurodiversity,
- evaluate a flow for exclusion and unnecessary burden.

## Key topics

- Accessibility
- Inclusion
- Human variation
- WCAG basics

- Error prevention
- Inclusive research

## Opening story structure

Open with a user who is not "disabled" in a narrow sense but is tired, stressed, distracted, using a phone in poor lighting, or reading unfamiliar language. Then expand into permanent, temporary, and situational variation.

## Inline check seeds

1. Which design choice creates the greatest cognitive burden?
2. Which issue would affect keyboard users?
3. Which message is written in the clearest plain language?
4. Which assumption excludes the most users?

## Guided practice

Learners complete a human variation audit: visual, motor, cognitive, emotional, linguistic, situational, assistive technology considerations.

## Module assessment focus

Learners identify accessibility risks and recommend improvements in a realistic product flow.
